Development activities

  • Openlava development

    In 2011 I initiated the development of www.openlava.org opensource distributed workload managemet system, I am the chief architect and developer of the openlava scheduling kernel and system protocols, and I act as the project's coordinator. The purpose of openlava is to provide the market with a open source version of Platform LSF™ and increase its world wide adoption as workload management tool.

  • PYX high performance multiprotocol software switch

    This project involves routing data between a user laptop and corporate server over a cryptography network. PYX is a fast data relay which makes possible to do secure online real time presentation across WAN. A worldwide cluster of PYX switches allows users to access the secure network from anywhere in the world wide wan. Pyx is at the core of MobiNET™

  • Project SMART

    Collecting data from a larger number, of user hard drives, being it a home system or a corporate network to build a large statistical database ofS.M.A.R.T. data to anticipate the failure of a given hard drive and backup its data.

  • External LSF plugins for large data set management

    In this activity I develop custom plugins for the LSF scheduler that allow jobs to be dispatched based on network and file system status. Jobs can be also scheduled to where the data are available by using so called data attractors.
